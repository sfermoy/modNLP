Query Language

Syntax:
word_1[+[[no_of_intervening]]word_2...].
Description:
The following options are currently supported by the keyword search box:
  • Single keywords: just enter type the word into the keyword box.
    Example:
    If you type test, for instance, the tool will retrieve all occurrences of the word test in the whole corpus (unless you restrict the search by selecting a sub-corpus). The search is case-insensitive by default (i.e. it will retrieve Test, TEST etc). You can specify case-sensitive search by ticking the appropriate box on the Options menu.
  • "Wildcards": this allows you to select prefixes.
    Example:
    Typing in test* will retrive all words wich start with test (e.g. test, tests, testament, etc).
  • "Regular expressions": this allows you to select any string that matches a regular expression (i.e. an element of a regular language). Regular expressions need to be enclosed in double quotation symbols (e.g. "regexp")
    Example:
    Typing in ".*less" will retrive all words wich start with test (e.g. test, tests, testament, etc).
  • sequences: this allows you to specify sequences os key words, and/or wildcards, and the maximum number of intervening words you wish to allow between each element in the sequence.

    Examples:

    • Entering seen+before will find ...never seen before... etc;
    • entering seen+[1]before finds, in addition, ...seen her before..., ...seen ie before..., and all sequences in which there is at most one word between seen and before.
    • Combinations of words and wildcards are also allowed, so entering know+before* will find ...know before..., ...know beforehand, etc.
